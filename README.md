# 🐾 Help a Paw – Animal Shelter Website

„Help a Paw” is a Frontend web application prototype designed as a university project for an animal shelter. The primary goal of the platform is to support stray animals by promoting adoptions, facilitating community involvement, and keeping users informed.

> ⚠️ **Project Functionality Note:** This project is a UI/UX presentation prototype. Forms (for adoption, volunteering, and donations) and their respective fields are implemented strictly for layout and visual demonstration purposes. It does not possess a backend or database to process real data.

---

## 📋 Platform Structure & Pages
The website is intuitively structured to help users find relevant information quickly, featuring the following pages:
* **Home Page** – Introduction to the shelter's mission and core purpose.
* **About Us** – The story, values, and team behind „Help a Paw”.
* **Dogs & Cats** – Dedicated sections showcasing a total of **16 animals** (8 dogs and 8 cats). Each animal features its own individual profile page with images and a short description.
* **Adoption Rules** – An informative guide outlining the necessary steps for a successful adoption.
* **Volunteering** – Information on how citizens can get involved, paired with a dedicated application form.
* **Contact** – The shelter's location details alongside a functional feedback form.

---

## 🛠️ Tech Stack
* **HTML5** – For semantic page structure and well-organized input forms.
* **CSS3** – For styling, typography, page layouts, and modern visual design.
* **JavaScript (Vanilla)** – For implementing interactive frontend components and dynamic UI elements.

---

## 📱 Fully Responsive Design & Adaptability
The user interface went through several iterations to guarantee a smooth browsing experience. It has been tested across multiple browsers and specifically optimized for various screen sizes:

* **Desktop:** The standard, full-featured version with complete top navigation bars and spacious layouts.
* **Tablets:** Margins and image dimensions were dynamically adjusted to prevent content cluttering.
* **iPad Pro:** A custom layout built specifically for intermediate, wider tablet screens, incorporating resized images and repositioned structural blocks.
* **Mobile Phones:** Content automatically flows into a single-column layout, and the top navigation is cleanly replaced by an interactive **hamburger menu**.

---

## 📈 The UI/UX Development Process
1. **Wireframing & Layout:** Mapping out the initial structure, navigation flow, and simple sketches to lock down key elements (menus, buttons, sections).
2. **Evolution & Tweaks:** Iteratively restructuring elements during development to create a clearer, more user-friendly flow.
3. **Evaluation:** Rigorous testing across multiple breakpoints and screens to ensure a stable, beautiful, and welcoming final interface.
